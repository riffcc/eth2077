# Strawmap Demolition Matrix

Goal: convert "beat the roadmap" into measurable, verifiable targets.

## Baseline vs ETH2077 Targets

| Dimension | Strawmap / ETH2030 direction | ETH2077 target posture |
|---|---|---|
| L1 scaling | Strawmap frames ~10x L1 scaling; ETH2030 frames 10k+ TPS | `>= 1,000,000` sustained L1 TPS with reproducible artifacts |
| Finality | ETH2030 frames ~18s | sub-5s deterministic finality with proof-linked safety |
| Node accessibility | low hardware / stateless direction | stateless-first architecture with low-cost participation target |
| Interoperability | explicit Strawmap objective | strict Ethereum API and execution compatibility |
| Robustness | explicit Strawmap objective | proof-gated safety/liveness plus adversarial fault validation |

## Non-Negotiable ETH2077 Claim Policy

1. No benchmark artifact -> no scaling claim.
2. No machine-checked theorem -> no safety claim.
3. Critical-path `axiom`/`sorry` present -> no "formally verified" claim.

## Operational Interpretation

"Demolish" means all three must hold:

1. Better headline numbers than roadmap baselines.
2. Better safety guarantees than narrative claims.
3. Better evidence quality than prototype-only implementations.
